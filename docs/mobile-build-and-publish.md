# Мобильные приложения — сборка и публикация

> Исходный код обоих приложений написан, но **на этом компьютере (Windows) он не собирался и не проверялся**. Это нормально: сборка iOS возможна только на Mac, а Android — в Android Studio. Ниже — что нужно и какие шаги, простыми словами.

Код лежит в репозитории:
- **Android** — папка `apps/android/` (Kotlin, Jetpack Compose).
- **iOS** — папка `apps/ios/` (Swift, SwiftUI).

Оба приложения общаются с тем же backend по REST API (контракт — `docs/superpowers/specs/2026-05-29-mobile-client-contract.md`).

## Android — как собрать и опубликовать (пошагово)

**А. Сборка и проверка**
1. Установить **Android Studio** (последнюю версию) + JDK 17 (идёт в комплекте).
2. Открыть папку `apps/android/` в Android Studio. Дождаться «Gradle Sync» (подтянет зависимости — нужен интернет).
3. Запустить на эмуляторе/телефоне (кнопка ▶). В разработке приложение ждёт API на `http://10.0.2.2:3000` (это «localhost» для эмулятора). Прогнать сценарии: вход по SMS → главный экран → чат → профиль. Зафиксировать мелкие правки, если что-то не собралось.
4. Перед боевым релизом поменять адрес API на ваш HTTPS-домен (в сетевом конфиге приложения).

**Б. Публикация в Google Play**
1. Зарегистрировать **аккаунт Google Play Developer** (play.google.com/console) — разовый взнос **$25**, верификация личности/организации.
2. Создать **ключ подписи** (keystore): в Android Studio `Build → Generate Signed Bundle/APK → Android App Bundle → Create new…`. **Сохранить keystore и пароли в надёжном месте** — без него нельзя выпускать обновления.
3. Собрать **release AAB**: `Build → Generate Signed Bundle/APK → Android App Bundle`, выбрать keystore, вариант `release`. Получится файл `.aab`.
4. В Google Play Console: **Create app** → название «VITTORIA HOME», язык, тип «App», бесплатное.
5. Заполнить обязательное: иконка (512×512), скриншоты (телефон), краткое и полное описание, **политика конфиденциальности** (URL — нужна из-за работы с ПДн), категория, контакты, анкеты «Content rating» и «Data safety» (указать сбор телефона/имени).
6. Загрузить `.aab` в трек **Internal testing** (для своих) → проверить → затем **Production** → **Submit for review**. Проверка обычно несколько часов–дней.

## iOS — как собрать и опубликовать (нужен Mac, пошагово)

**А. Сборка и проверка (на Mac)**
1. Установить **Xcode** (из App Store) и **XcodeGen**: `brew install xcodegen`.
2. `cd apps/ios && xcodegen generate` — создаст `VittoriaClient.xcodeproj`.
3. Открыть проект в Xcode → в `Signing & Capabilities` выбрать вашу команду (Apple Developer). Запустить на «iOS Simulator». Прогнать сценарии (как у Android). Зафиксировать мелкие правки SDK-версий, если всплывут.
4. Перед релизом поменять адрес API на боевой HTTPS-домен.

**Б. Публикация в App Store**
1. Оформить **Apple Developer Program** (developer.apple.com) — **$99/год**, верификация.
2. В **App Store Connect** (appstoreconnect.apple.com) создать приложение: Bundle ID `app.vittoria.client`, название «VITTORIA HOME».
3. В Xcode выбрать устройство «Any iOS Device», затем `Product → Archive`. В окне Organizer → **Distribute App → App Store Connect → Upload**.
4. В App Store Connect заполнить карточку: иконка (1024×1024), скриншоты (нужных размеров), описание, ключевые слова, возрастной рейтинг, **политика конфиденциальности** (URL) и анкета «App Privacy» (сбор телефона/имени).
5. Выбрать загруженный билд → **Submit for Review**. Ревью Apple обычно 1–3 дня (бывает дольше). Часто просят демо-доступ — дать тестовый телефон + код.

## Важно про «всё должно работать»

- **Что уже работает и можно показать сейчас:** backend + веб-панель (admin и кабинет партнёра) — они запускаются на этом ПК, вы их видели.
- **Мобильные приложения:** код готов, но «работают» они станут после первой сборки на Mac (iOS) / в Android Studio (Android). Почти наверняка при первой сборке всплывут мелкие правки (версии библиотек, подпись) — это нормальный этап для любого мобильного проекта. После сборки нужно прогнать по сценариям: вход по SMS → главный экран с заказом → чат → профиль.
- **amoCRM:** ✅ уже подключён (реальный режим `http`, токен в `.env`, воронка «VITTORIA HOME» и служебные поля созданы). Входящие вебхуки заработают после деплоя на публичный домен.
- **Реальные SMS:** ⏳ нужны логин/пароль SMSC.ru → переключение SMS в режим `smsc`. Сейчас код подтверждения печатается в консоль сервера (dev-режим).
