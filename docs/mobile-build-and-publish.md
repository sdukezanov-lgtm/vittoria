# Мобильные приложения — сборка и публикация

> Исходный код обоих приложений написан, но **на этом компьютере (Windows) он не собирался и не проверялся**. Это нормально: сборка iOS возможна только на Mac, а Android — в Android Studio. Ниже — что нужно и какие шаги, простыми словами.

Код лежит в репозитории:
- **Android** — папка `apps/android/` (Kotlin, Jetpack Compose).
- **iOS** — папка `apps/ios/` (Swift, SwiftUI).

Оба приложения общаются с тем же backend по REST API (контракт — `docs/superpowers/specs/2026-05-29-mobile-client-contract.md`).

## Android — как собрать и опубликовать

1. Установить **Android Studio** (последнюю версию).
2. Открыть папку `apps/android/` в Android Studio. Дождаться «Gradle Sync» (подтянет зависимости).
3. Запустить на эмуляторе или телефоне (кнопка ▶). В режиме разработки приложение ждёт API по адресу `http://10.0.2.2:3000` (это «localhost» с точки зрения эмулятора). Перед боевым релизом адрес меняется на ваш HTTPS-домен в `app/build.gradle.kts` / конфиге сети.
4. Для публикации: создать **ключ подписи** (keystore), собрать «release AAB» (`Build → Generate Signed Bundle`), завести приложение в **Google Play Console** ($25), загрузить AAB, заполнить карточку (иконки, скриншоты, описание, политика конфиденциальности) и отправить на проверку.

## iOS — как собрать и опубликовать (нужен Mac)

1. На Mac установить **Xcode** и инструмент **XcodeGen**: `brew install xcodegen`.
2. В терминале: `cd apps/ios && xcodegen generate` — создаст файл проекта `VittoriaClient.xcodeproj`.
3. Открыть его в Xcode, в настройках Signing выбрать вашу команду (Apple Developer аккаунт), запустить на «iOS Simulator». В разработке API берётся с `http://localhost:3000`.
4. Для публикации: в Xcode `Product → Archive`, загрузить через **App Store Connect** (Apple Developer $99/год), заполнить карточку (иконки, скриншоты, описание, политика конфиденциальности) и отправить на ревью Apple.

## Важно про «всё должно работать»

- **Что уже работает и можно показать сейчас:** backend + веб-панель (admin и кабинет партнёра) — они запускаются на этом ПК, вы их видели.
- **Мобильные приложения:** код готов, но «работают» они станут после первой сборки на Mac (iOS) / в Android Studio (Android). Почти наверняка при первой сборке всплывут мелкие правки (версии библиотек, подпись) — это нормальный этап для любого мобильного проекта. После сборки нужно прогнать по сценариям: вход по SMS → главный экран с заказом → чат → профиль.
- **Реальные SMS и amoCRM:** для боевого режима нужны (1) логин/пароль SMSC.ru и переключение SMS в режим `smsc`; (2) токен доступа amoCRM и переключение синхронизации в режим `http`. Сейчас всё в безопасном «dev/mock» режиме.
