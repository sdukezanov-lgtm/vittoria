## Что ОСТАЛОСЬ (следующие шаги) ⏳
1. **Деплой на сервер** — нужен VPS (Ubuntu) + домен (поддомены `api.` и `admin.`). Рецепт: `docs/deployment-guide.md`. После деплоя — прописать вебхук amoCRM (мгновенная синхронизация).
2. **Реальные SMS** — доступы SMSC уже есть (в `.env`/`.env.prod`), переключить `SMS_PROVIDER_MODE=smsc`. Локально из этой среды отправку проверить не удалось (smsc.ru недоступен) — проверим на сервере.
3. **Мобильные: сборка + публикация** — Mac+Xcode (iOS), Android Studio (Android); аккаунты Apple ($99/год) и Google ($25). Гайды: `mobile-build-and-publish.md`, `store-accounts-and-publishing.md`.
4. **Push-уведомления** — ключи FCM (Android) и APNs (iOS), появляются на этапе мобильной сборки.
5. **Убрать тестовые данные в amoCRM** — вручную удалить сделку и контакт «ТЕСТ … (удалить)» в аккаунте (API amoCRM не даёт удалять).

- ⚠️ `.env` НЕ в Git. Это правильно (секреты не уезжают в открытый репозиторий). Бэкап — скопировать файл в надёжное место.

## Как запустить локально (демо)
```
pnpm dev:infra            # база, redis, хранилище (Docker)
pnpm --filter @vittoria/api prisma:migrate:deploy   # один раз
pnpm dev:api              # сервер на :3000
pnpm dev:admin            # панель на :5173
```
Вход админа: телефон `+79990000000`; код в dev-режиме печатается в консоль сервера (`[DEV-SMS] ... HOME: XXXX`). Тестовый партнёр: `+79991110000`. Тестовый клиент с заказами: `+79991234567`.
