# VITTORIA HOME — единый бренд-каркас, кабинет с боковым меню, отполированные компоненты (дизайн-спека)

Дата: 2026-06-02
Статус: согласовано в брейншторме, ожидает плана реализации
Родительская спека: [`2026-06-01-vittoria-web-client-cabinet-and-rebrand-design.md`](2026-06-01-vittoria-web-client-cabinet-and-rebrand-design.md)

---

## 1. Контекст

Веб-часть (кабинет клиента + перекраска админки) уже реализована в ветке
`feature/web-client-cabinet-rebrand` по спеке 06-01 и запускалась на localhost.
Заказчик прислал **уточнённый скриншот** веб-панели в новом стиле. Сверка скриншота
с текущим кодом показала: **скриншот отполированнее, чем реально собрано**. На скриншоте есть
элементы, которых в коде сейчас нет:

- монограмма «VH» (засечная, золото+графит) слева от слова «VITTORIA HOME»;
- иконки у пунктов бокового меню;
- золотая левая полоска + бежевый фон активного пункта меню;
- иконки внутри бейджей этапов;
- прогресс не тонким баром, а **плашкой с числом «NN%» внутри**, окрашенной по этапу
  (золото в работе, голубой/прочие — по цвету этапа, зелёный — 100 %);
- крупный засечный заголовок внутри большой белой карточки.

Скриншот трактуется как **макет-ориентир** для всей веб-панели, а не как текущий вид.

### Решения брейншторма (2026-06-02)

1. **Раскладка кабинета клиента** — **боковое меню** с иконками (как на скриншоте админки),
   а не верхняя шапка без меню (текущий `ClientLayout`).
2. **Охват** — отполированные элементы применяем **и к кабинету, и к админке**
   (и к партнёрскому кабинету), чтобы все разделы выглядели одинаково и правились в одном месте.

### Цель

Собрать единый бренд-каркас и общие компоненты, довести вид всех веб-разделов до скриншота-ориентира.
Меняется **только внешний вид** — backend, API, бизнес-логика, данные, входы и маршруты не трогаем.

## 2. Не-цели

- Не меняем backend, REST API, схему БД, бизнес-логику, интеграции, состав данных.
- Не делаем реальные фото изделий — остаётся `ProductPlaceholder`.
- Не меняем набор пунктов меню админки/партнёра и маршруты (только их оформление + иконки).
- Не трогаем мобильные приложения и деплой (отдельные этапы).
- Не переписываем чат (`Conversation`) — переиспользуется как есть.

## 3. Архитектурный подход

Вариант **A** (выбран): единый конфигурируемый каркас `BrandShell` + общие бренд-компоненты;
все три роли (admin / partner / client) строятся поверх него, передавая свой список пунктов меню.
Отвергнуты: B (править каждый layout отдельно — дублирование и риск расхождения),
C (только тема Mantine — не умеет конфигурировать меню/иконки, полумера).

Стек прежний: React 18 + TS + Vite + Mantine 7 + React Query + React Router, приложение `apps/admin`.

## 4. Общий бренд-слой (`apps/admin/src/brand/`)

### 4.1 `Logo` (обновить)
Добавить засечную монограмму «VH» (Cormorant, золото `BRAND.gold` + графит `BRAND.graphite`)
слева от текста «VITTORIA HOME». Параметры: `size`, `tagline`, опц. `monogramOnly` (для узких мест).
Монограмма — лёгкая, на чистом CSS/тексте (без растровых ассетов).

### 4.2 `BrandShell` (новый) — каркас страницы
```ts
type NavItem = { to: string; label: string; icon: ReactNode; badge?: ReactNode };
type BrandShellProps = { nav: NavItem[]; headerRight?: ReactNode };
```
- `AppShell` (header 56, navbar 220, padding md, мобильный бургер).
- **Шапка:** бургер (моб.), `<Logo size={22} />` (с монограммой), справа — `headerRight`
  (телефон/контакт + кнопка «Выход»).
- **Навбар:** `nav.map` → `NavLink` (`component={RouterNavLink}`), `leftSection={icon}`,
  опц. `rightSection={badge}`. Активный пункт: бежево-золотой фон (Mantine active, variant light/`gold`)
  + **золотая левая полоска** (через `styles`/CSS-правило по `data-active`).
- **Контент:** `AppShell.Main` → `<Outlet />` на фоне `BRAND.bg`.

### 4.3 `StageBadge` (обновить)
Добавить `leftSection` — иконку этапа из карты `STAGE_ICON` (см. §7). Цвет (`STAGE_COLOR`),
текст (`STAGE_LABELS`), `variant="light"`, `radius="sm"` — сохранить.

### 4.4 `ProgressPill` (новый)
```ts
type ProgressPillProps = { percent: number; color?: string; width?: number };
```
Плашка фикс. ширины: светлая подложка (тинт `color`), заливка по `percent`, число «NN%» внутри
по центру. `color` по умолчанию `'gold'`; в таблицах передаётся `STAGE_COLOR[stage]`
(100 % → зелёный). Заменяет связку `<Progress/> + <Text/>` в таблицах.

### 4.5 `PageCard` (новый, тонкая обёртка)
```ts
type PageCardProps = { title: ReactNode; actions?: ReactNode; children: ReactNode };
```
Большая белая карточка (`radius="lg"`, мягкая тень, `p="lg"`) с засечным заголовком сверху
(`Title order={2}`) и опц. блоком действий. В неё кладут контент страниц-списков
(как блок «Заказы» на скрине).

### 4.6 `OrderStatusStepper` (обновить, бонус)
Кружки этапов показывают **иконку этапа** (`STAGE_ICON`) вместо номера:
done/active — золотая заливка + белая иконка; upcoming — серый + приглушённая иконка.

## 5. Кабинет клиента — боковое меню

`ClientLayout` переписывается на `BrandShell`. Пункты меню:

| Иконка | Пункт | Маршрут |
|---|---|---|
| `IconHome` | Главная | `/cabinet` |
| `IconPackage` | Мои заказы | `/cabinet/orders` |
| `IconMessageCircle` | Чат | `/cabinet/chat` (активный заказ) |
| `IconUser` | Профиль | `/cabinet/profile` |

`headerRight` кабинета: телефон сервисного отдела (`service/contact`, кликабельный) + «Выход».

### Страницы
- **Главная** (`/cabinet`) — дашборд активного заказа: `OrderSummaryCard` + `OrderStatusSection`
  (с обновлённым степпером) + блок быстрых ссылок. Если заказов несколько — компактный переключатель
  (чипы) сверху; выбранный заказ считается «активным».
- **Мои заказы** (`/cabinet/orders`) — список заказов клиента карточками (договор, изделие,
  `StageBadge`, `ProgressPill`); клик → Главная этого заказа (`/cabinet/orders/:id` либо выбор активного).
- **Чат** (`/cabinet/chat`, `/cabinet/chat/:id`) — переписка по активному заказу
  (`Conversation`); если заказов несколько — переключатель/выбор заказа.
- **История этапов** (`/cabinet/history/:id`) — вертикальный таймлайн (как сейчас), доступна с Главной.
- **Профиль** (`/cabinet/profile`) — имя/фамилия/телефон, выход.

Страницы кабинета оборачиваются в `PageCard` (где уместно), используют `StageBadge` (с иконкой)
и `ProgressPill`. Маршруты `/cabinet/*` остаются под `RoleGate allow={['client']}`.

> Примечание по навигации: «Чат» и «История» по природе привязаны к заказу (`:id`). Пункт меню
> «Чат» ведёт к чату активного (выбранного на Главной/в «Мои заказы») заказа; при единственном
> заказе — сразу к нему. Это решается выбором активного заказа на фронте, без изменения API.

## 6. Перекраска админки и партнёрского кабинета

- **`AppLayout` (админ)** → на `BrandShell`. Пункты с иконками:
  Заказы `IconShoppingBag` (`/orders`), Чат `IconMessageCircle` (`/chats`, бейдж непрочитанных),
  Партнёры `IconUsers` (`/partners`), Комиссии `IconPercentage` (`/commissions`),
  Аудит `IconShieldCheck` (`/audit`), Шаблоны `IconFileText` (`/templates`).
  `headerRight`: телефон пользователя + «Выход».
- **`PartnerLayout` (партнёр)** → на `BrandShell`. Пункты:
  Мои заказы `IconShoppingBag` (`/partner/orders`), Мои вознаграждения `IconCoin`
  (`/partner/commissions`), Профиль `IconUser` (`/partner/profile`).
- **`OrdersPage` (админ)** → обернуть в `PageCard` («Заказы», поиск+фильтр в actions),
  колонку «%» — на `ProgressPill` (цвет = `STAGE_COLOR[stage]`). `StageBadge` уже с иконкой.
- **`PartnerOrdersPage`** → привести к тому же виду: сейчас там голый текст этапа и «NN%»;
  заменить на `StageBadge` (с иконкой) + `ProgressPill`, обернуть в `PageCard`.
- **`OrderPage` / `PartnerOrderPage`** (детали заказа) → этап/прогресс через общие компоненты,
  засечные заголовки, белые карточки (косметика, без изменения данных).
- **`LoginPage`** → монограмма появится через обновлённый `Logo`; добавить тёплый фон/акцент карточки.

## 7. Иконки (`@tabler/icons-react`)

Добавить зависимость `@tabler/icons-react` (штатный набор для Mantine, tree-shakeable).

Карта `STAGE_ICON: Record<OrderStage, Icon>`:

| Этап | Иконка |
|---|---|
| `preparation_for_production` (Подготовка для производства) | `IconBox` |
| `detailing` (Деталировка) | `IconZoomCode` / `IconRulerMeasure` |
| `materials_arrival` (Поступление материалов) | `IconTruckDelivery` |
| `production` (Производство изделия) | `IconTool` |
| `transfer_to_warehouse` (Передача на склад) | `IconBuildingWarehouse` |
| `completeness_check` (Проверка комплектности) | `IconClipboardCheck` |
| `ready_for_delivery` (Готовность к передаче) | `IconCircleCheck` |

(Конкретные иконки финализируются в коде по визуальной сверке; карта — единый источник.)
