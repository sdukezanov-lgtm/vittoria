## Как снова запустить localhost (демо)
> ⚠️ Нюанс: порт 5432 занят контейнером **`arbitros-db-1`** (другой проект). Поэтому нашу базу
> поднимали на **5433** и запускали API с переопределённым `DATABASE_URL` (без правки `.env`).

```bash
# 1. Инфраструктура (наш postgres на 5433, чтобы не конфликтовать с arbitros-db на 5432):
POSTGRES_PORT=5433 docker compose -f infra/docker-compose.dev.yml up -d
# 2. API на :3000 с базой 5433:
DATABASE_URL='postgresql://vittoria:vittoria@localhost:5433/vittoria_dev' pnpm dev:api
# 3. Веб-панель на :5173:
pnpm dev:admin
```
Если порт 5432 уже свободен — можно штатно: `pnpm dev:infra` + `pnpm dev:api` (без override).

Открыть: **http://localhost:5173**

**Тестовые входы** (SMS не шлются; код печатается в КОНСОЛЬ API — попроси ассистента глянуть и продиктовать):
- Клиент с 3 заказами: **`+79991234567`** → кабинет (изобр.1).
- Админ: **`+79990000000`** → панель (изобр.2).
- Партнёр: `+79991110000`.
